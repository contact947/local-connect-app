# LocalConnect - モバイルアプリUI/UX設計書

## デザインコンセプト

**ターゲット**: 17〜24歳の若者（高校生・大学生・新社会人）
**デザイン方針**: Z世代向けのシンプルで直感的なUI、SNSアプリに近い操作感、余白を活かした洗練されたデザイン

## カラーパレット

- **Primary**: `#0a7ea4` - アクセントカラー（ボタン、アクティブ状態）
- **Background**: Light `#ffffff` / Dark `#151718`
- **Surface**: Light `#f5f5f5` / Dark `#1e2022` - カード背景
- **Foreground**: Light `#11181C` / Dark `#ECEDEE` - メインテキスト
- **Muted**: Light `#687076` / Dark `#9BA1A6` - サブテキスト

## 画面一覧

### 1. ホームタブ（Home）
**目的**: ユーザーの地域に特化したニュースとコラムを表示
**主要コンテンツ**:
- ユーザーの登録住所に基づく地域ニュース（アクセス数順）
- 地域コラム記事
- 各記事カード: サムネイル画像、タイトル、カテゴリタグ、投稿日時
**レイアウト**:
- 縦スクロール可能なフィード形式
- カード型デザイン（角丸、シャドウ）
- タップで詳細画面へ遷移

### 2. ニュースタブ（News）
**目的**: 全国のニュース記事を閲覧
**主要コンテンツ**:
- 全国ニュース一覧（新着順）
- カテゴリフィルター（店舗・イベント・インタビュー等）
- 記事詳細画面: 本文、画像、関連記事
**レイアウト**:
- リスト形式またはグリッド形式
- 検索バー（上部）
- カテゴリタブ切り替え

### 3. イベントタブ（Events）
**目的**: イベント参加とチケット管理
**主要コンテンツ**:
- **参加予定セクション**: チケット発行済みイベント
  - イベント名、日時、会場
  - QRコード表示ボタン
- **地域イベントセクション**: 登録住所に基づくイベント一覧
  - イベントカード: サムネイル、タイトル、日時、価格
- **イベント詳細画面**:
  - イベント情報（説明、会場、日時、価格）
  - チケット購入ボタン
- **チケット購入フロー**:
  - 枚数選択 → 決済画面（モック） → QRコード発行
**レイアウト**:
- タブ切り替え（参加予定 / 地域イベント）
- カード型デザイン
- QRコード: フルスクリーン表示、明るさ自動調整

### 4. ギフトタブ（Gifts）
**目的**: 近隣店舗のギフト・クーポンを表示
**主要コンテンツ**:
- GPS位置情報に基づく近い順のギフト一覧
- ギフトカード: 店舗名、特典内容、距離、有効期限
- 利用条件表示（年齢制限、学校区分等）
- 使用済み/未使用ステータス
**レイアウト**:
- 縦スクロールリスト
- 距離表示（例: 500m）
- 使用ボタン → QRコード表示

### 5. アカウント画面（Account）
**目的**: ユーザー情報管理
**主要コンテンツ**:
- プロフィール情報表示・編集
  - 名前
  - 年齢
  - 性別
  - 住所（地域判定に使用）
  - アカウントID（自動付番、表示のみ）
- ログアウトボタン
**レイアウト**:
- フォーム形式
- 編集モード切り替え
- 保存ボタン

## 主要ユーザーフロー

### フロー1: イベントチケット購入
1. ホーム or イベントタブからイベントを探す
2. イベントカードをタップ → 詳細画面
3. 「チケットを購入」ボタンをタップ
4. 枚数選択
5. 決済画面（モック）で確認
6. 購入完了 → QRコード自動発行
7. 「参加予定」タブにイベント追加
8. 入場時にQRコードを表示

### フロー2: ギフト利用
1. ギフトタブを開く（GPS位置情報取得）
2. 近い順にギフト一覧表示
3. 利用したいギフトをタップ
4. 詳細確認（利用条件、有効期限）
5. 「使用する」ボタンをタップ
6. QRコード表示
7. 店舗でQRコードをスキャン
8. 使用済みステータスに変更

### フロー3: 地域ニュース閲覧
1. ホームタブを開く（登録住所に基づく地域ニュース表示）
2. 記事カードをタップ
3. 記事詳細画面で本文閲覧
4. 関連記事へ遷移可能

## デザイン原則

### モバイルファースト（縦向き 9:16）
- 片手操作を前提としたUI配置
- 重要なボタンは画面下部または中央
- タップターゲットは最小44x44pt

### iOS HIG準拠
- ネイティブコンポーネントの活用
- システムフォント使用
- 標準的なナビゲーションパターン

### シンプル & 直感的
- 階層は最大3レベルまで
- アクションは1タップで完結
- 明確なCTA（Call to Action）

### アクセシビリティ
- 十分なコントラスト比
- 読みやすいフォントサイズ（最小14pt）
- ダークモード対応

## アニメーション & フィードバック

- **タップフィードバック**: 軽い触覚フィードバック（Haptics.Light）
- **画面遷移**: スライドアニメーション（250ms）
- **カード表示**: フェードイン（200ms）
- **ボタン押下**: スケール 0.97 + 透明度 0.9

## 技術的考慮事項

- **認証**: Manus-OAuth使用
- **データ取得**: 登録住所ベース（ホーム、ニュース、イベント）
- **位置情報**: ギフトタブのみGPS使用（expo-location）
- **QRコード**: expo-barcode-scanner / react-native-qrcode-svg
- **決済**: モック実装（実際の課金なし）
- **通知**: expo-notifications（イベント前日リマインダー等）
